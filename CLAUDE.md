# CLAUDE.md — 작업 헌법 (최우선 준수)

> 이 문서의 규칙은 작업 시작 시 항상 먼저 읽고, 숨 쉬듯 지켜야 할 최우선 지침이다.

## 1. 프로젝트 방향
- 모바일 앱으로 옮길 것을 전제로 만든다.
- Next.js, 링크로 바로 사용 가능하게.
- 코드를 쓰기 전에 항상 먼저 제안한다: 화면 흐름 / 주요 기능 / 파일 구조.
- 화면은 세로형 폰(모바일) 기준으로 먼저 설계한다.

## 2. 구조 원칙
- 게임 로직, UI, 데이터 저장 코드를 가능한 한 분리한다.
  - 한 파일에 몰지 않기 (단, MVP 단계에서 과한 분할은 NO)
- 임시 구현이라도 나중에 교체하기 쉽게 작게 만든다.
- 새 기능을 만들 때 기존 구조를 섞어 망가뜨리지 말고, 필요한 파일을 명확히 분리한다.
- 전역 CSS·공통 컴포넌트 수정 NO. 전용 className 사용은 OK.

## 3. 수정 규칙
- 수정 시, 수술 부위만 칼 대기 (타 로직 수정 절대 금지).
- 다른 코드를 수정해야만 하는 상황이 오면, 무작정 고치지 말고 먼저 아래 4가지를 보고하고 허락을 받는다:
  1. 이해 (왜 필요한지)
  2. 수정 파일
  3. 안 건드릴 파일
  4. 영향 범위
  - "진행해" 라는 응답을 받으면 그때 수정 시작.

## 4. 작업 방식 (탑다운, Big Block 우선)
- 탑다운(Top-down) 큰 그림 중심 작업.
- 처음부터 코드를 잘게 쪼개거나(TDD, 깊은 모듈화), 복잡한 구조를 만들어서 일을 벌이지 않는다.
- 먼저 크고 직관적인 큰 덩어리 구조(Big Block)로 전체 흐름이 구동되는 것부터 눈으로 확인시킨다.
- 디테일한 쪼개기는 흐름이 성공한 뒤에 진행한다.
- 한 화면/기능씩 만들고 확인 후 다음 단계로 넘어간다.
- 기능마다 "성공 기준"을 한 줄로 정의한다.

## 5. 커밋 & 변경 관리
- 큰 구조 변경 전에는 먼저 알려준다 (사용자가 직접 커밋함).
- 중요 로직에는 짧은 한글 주석을 남긴다.

## 6. 진행 신호
- 모든 제안은 "진행해" 라는 응답을 받은 뒤에 코딩을 시작한다.
